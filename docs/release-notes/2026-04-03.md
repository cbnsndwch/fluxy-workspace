## Overview

This release delivers a complete WhatsApp channel integration. Sebastian can now communicate bidirectionally over WhatsApp — including voice messages and audio — with proper channel/business logic separation, a typing indicator, and a new chat bubble that lets users link their WhatsApp directly from the conversation UI.

---

## New Features

### WhatsApp Channel (`supervisor/channels/`)

Full WhatsApp integration powered by `@whiskeysockets/baileys`. Two operating modes:

- **Channel mode** — attaches to your own number; only self-chat triggers the agent (everything else is silently ignored)
- **Business mode** — uses a dedicated Fluxy number; admin list gets the main system prompt, everyone else gets the active skill's script

The channel layer lives at `supervisor/channels/whatsapp.ts` with a unified `ChannelManager` (`manager.ts`) and a shared `types.ts` definition so future channels (Telegram, etc.) plug in cleanly.

### Voice Message & Audio Support

Incoming WhatsApp voice notes and audio messages are now processed by Sebastian. The manager decodes audio attachments and passes them through the agent pipeline.

### Typing Indicator

Sebastian sends a WhatsApp `typing...` presence signal while composing a response, so the conversation feels live rather than appearing to hang.

### WhatsApp Linking Bubble

A new button in the chat bubble lets users initiate the WhatsApp QR link flow directly from the chat UI — no need to visit a separate setup page. The `MessageBubble` component in the chat supervisor was updated to render this action.

### Channel / Business Logic Separation

`supervisor/fluxy-agent.ts` was refactored to cleanly distinguish:
- **Admin messages** (via `role=admin`) → full Sebastian system prompt, all capabilities
- **Customer messages** (via `role=customer`) → active skill's `SCRIPT.md` as system prompt

The shared config (`shared/config.ts`) was updated to carry channel metadata and the worker system prompt was updated to document this routing behavior.

### Built-in Skills Updated

Four built-in skills were added/refreshed:
- `whatsapp-support` — customer-facing support script
- `code-reviewer` — code review workflows
- `daily-standup` — standup facilitation
- `workspace-helper` — general workspace assistant
