# 2026-03-30 — PWA Polish, Chat UX & Push Notifications via WebSocket

> **Commits:** `97d753d` → `ca3155e` (2026-03-30 – 2026-03-31)

## Overview

A batch of UX and PWA polish: a spray animation on the chat bubble, white splash screen fix, `.mov` video support, push notification subscription moved to WebSockets, and a Vite 8 upgrade.

---

## New Features

### Spray Animation on Chat Bubble

A sprite-sheet based animation plays on the chat bubble when it first appears. The animation uses a `spritesheet.webp` asset and is rendered via `supervisor/widget.js`. Adds a small delightful moment to first impressions.

### Push Notifications via WebSocket

Push notification subscribe/unsubscribe flows were moved from HTTP to WebSocket messages. This means:
- Notifications register immediately when the WS connection is established
- No extra HTTP round-trips for subscription management
- Notifications work correctly through Cloudflare tunnels (which previously had issues with HTTP subscription endpoints)

### `.mov` Video Support

Safari/iOS-generated `.mov` video files can now be sent in chat. A conversion pipeline handles the `.mov` → `.webm`/`.mp4` fallback for non-Safari browsers. Animation assets (`fluxy_say_hi`, `fluxy_happy`, `fluxy_happy_reappearing`) were also converted to `.webm`/`.mp4`.

### Init Wallet Creation

New users who go through onboarding now have a wallet record initialized automatically. This lays the groundwork for future billing/credits features.

### Handle Validation

Handle codes are validated earlier in the onboard flow to prevent invalid handles from being set.

## Fixes

### White Splash Screen Fixed

The PWA was showing a white flash before the chat bubble loaded. The service worker (`workspace/client/public/sw.js`) and entry point were updated to eliminate the blank intermediate state.

### Update CLI Fix

`cli/commands/update.ts` was updated to correctly trigger the `.update` file watcher in the new workspace layout (path had shifted after the workspace restructure).
