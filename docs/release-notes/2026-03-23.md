## Overview

A large-scale architecture refactor touching nearly every layer of Fluxy. The headline changes: React Router replaces the flat single-page setup, the workspace directory is no longer hardcoded, the build system migrated to pnpm, and the entire supervisor/worker/CLI/skills stack was modernized.

---

## Architecture Changes

### React Router for Workspace Frontend

The workspace frontend now uses proper client-side routing via `react-router`. A `Vite` config entry was added to resolve the React Router package correctly, and the workspace `App.tsx` was restructured around `<Routes>` + `<Route>` components.

**Why it matters:** Previously, the workspace was a single flat page. Now each app (CRM, Research, Analytics, etc.) lives at its own URL (`/crm`, `/research`, `/analytics`…), enabling browser back/forward, deep links, and route-level data loaders.

### Dynamic Workspace Directory

The Supervisor, Vite, and shared utilities previously had the workspace path hardcoded. This was changed to read from an environment variable (`WORKSPACE_DIR`), making Fluxy installable anywhere on disk without code changes.

Changes:
- `supervisor/index.ts` — uses dynamic path for spawning workspace backend
- `vite.config.ts` — resolves workspace via env var, not `__dirname`
- `shared/` — logs the resolved workspace directory on initialization

### Supervisor & Worker Modules Refactored

Major internal restructure of `supervisor/` and `worker/`:
- Modules split into more focused files
- API surface cleaned up
- Shared utilities extracted to `shared/`

### CLI Refactored

The `bin/cli.js` entry point and `cli/` commands were overhauled:
- New command structure
- Better error handling
- Consistent output formatting

### Skills Architecture Updated

`skills/` paths and the built-in skill manifests were updated to match the new workspace layout. Skill `SKILL.md` files and `.claude-plugin/plugin.json` descriptors refreshed for all built-in skills.

## Build System

### Migrated to pnpm

The project migrated from npm to **pnpm** as the package manager:
- `pnpm-workspace.yaml` added for monorepo workspace config
- `build: update build configuration` commit migrated lockfile
- pnpm's stricter hoisting rules exposed some implicit dependency issues (resolved in same batch)

### oxlint + oxfmt

**oxlint** (fast Rust-based ESLint alternative) and **oxfmt** (formatter) configuration files were added. These run locally for fast lint feedback during development.

### Docker Updates

Docker Compose configuration updated to reflect new workspace directory handling and pnpm package manager.
