# 2026-04-01 — Supervisor v2, New Workspace Design & Password Reset CLI

> **Commits:** `b1a7fdf` → `ba6a358` (2026-04-01)
> **Author:** Bruno Bertapeli

---

## Overview

A significant infrastructure release. The separate worker process was eliminated — everything now runs inside the Supervisor. The workspace got a new design (React Router, updated layout, health checker), and a new `fluxy password-reset` CLI command was added for account recovery.

---

## Architecture Changes

### Worker Process Eliminated

The standalone `worker/` process that previously ran alongside the Supervisor has been removed. All worker functionality — API handling, database access, agent invocation — now lives directly inside `supervisor/index.ts` and `supervisor/worker.ts`.

**Before:** `fluxy start` spawned: Supervisor + Worker + Workspace backend
**After:** `fluxy start` spawns: Supervisor + Workspace backend

This simplifies process management, reduces startup time, and eliminates a class of IPC errors.

### Supervisor Backend Restart Fix

A bug where the Supervisor wouldn't reliably restart the workspace backend after file changes was fixed in `supervisor/backend.ts`. The restart now fires correctly on `.ts`, `.js`, and `.json` changes.

### Workspace Package JSON

`workspace/package.json` was added — the workspace now has its own npm manifest, separate from the root `package.json`. This enables the workspace to manage its own dependencies independently. `workspace/.npmrc` was also added to configure the private registry scope.

The `postinstall.js` script was updated to bootstrap the workspace's own `node_modules` on first install.

## Workspace Redesign

### React Router for Workspace

The workspace frontend now uses React Router for client-side navigation. All page components are registered as proper routes, enabling:
- Clean browser history (back/forward works)
- Deep-linkable URLs
- Loader functions for data prefetching

### New Layout Components

- `DashboardLayout` — top-level shell with sidebar + content area
- `DashboardPage` — updated dashboard with widget grid
- `Footer` — new workspace footer
- `MobileNav` — mobile navigation bar
- `Sidebar` — updated sidebar with React Router `NavLink` (active state, smooth transitions)
- `WorkspaceTour` — onboarding tour component for new workspaces

### System Prompt Updates

The Fluxy system prompt was updated to document the new workspace structure, routing conventions, and the fact that the worker process was consolidated.

### Workspace Health Check

A health check endpoint and dashboard widget were added. The supervisor checks workspace backend health on startup and periodically, surfacing any issues.

## New CLI Command: `fluxy password-reset`

```
fluxy password-reset
```

Clears the portal password and re-triggers the onboarding wizard on next access. Useful for:
- Forgotten passwords (without needing SSH access to the config)
- Transferring a Fluxy instance to a new owner
- Recovery after lockout

Implementation: `cli/commands/password-reset.ts` — reads the worker DB, wipes `portal_pass`, sets a flag that redirects the next visitor to the onboarding wizard.
