## Overview

Two targeted improvements: a WebSocket proxy for workspace API calls (fixes POST/PUT failures through Cloudflare tunnels), and additional npm packages required by the workspace template.

---

## New Features

### WebSocket Proxy for Workspace API Calls

**Problem:** When accessing Fluxy through a Cloudflare tunnel, `POST`/`PUT`/`DELETE` requests to `/app/api/*` would intermittently fail. Cloudflare's tunnel layer strips some HTTP headers and imposes constraints that break non-GET requests.

**Solution:** All `/app/api/*` calls from the workspace frontend are now automatically proxied through the existing WebSocket connection when available. The `app-ws.js` script intercepts `fetch()` calls matching `/app/api/*`, tunnels them as WebSocket messages, and delivers the response — fully transparent to the frontend code.

**Effect:** No changes needed in frontend fetch calls. `fetch('/app/api/whatever')` just works, whether accessed locally or through Cloudflare. The proxy falls back to regular HTTP if the WS connection is unavailable.

### Workspace Template Packages

Additional npm packages required by the workspace starter template were added to `package.json`. These are needed for the workspace's own frontend/backend to function out of the box without a manual `npm install` step.
