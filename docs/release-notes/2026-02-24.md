## Overview

New URL routing style adopted across the platform, and the backend/frontend hot reload loop was stabilized — making development significantly smoother.

---

## New Features

### New Domain Routing Style

The URL scheme for accessing Fluxy instances was redesigned. The new style (`EVERYTHING WORKING WITH NEW DOMAIN STYLE`) established the routing conventions that everything since has been built on:
- Each Fluxy instance is accessed at `fluxy.bot/<handle>` (or self-hosted equivalent)
- The supervisor routes requests to the correct instance
- The workspace is served at `/app/`

This was the commit that locked in the pattern: the supervisor strips the `/app` prefix before forwarding to the workspace backend, which means frontend fetch calls use `/app/api/...` while backend Express routes register as `/api/...`.

### Backend + Frontend Vite Hot Reload

The workspace backend (Express/TypeScript) and frontend (Vite/React) now restart and refresh reliably on file changes during development:
- Backend: file watcher triggers restart, `.backend.log` captures output
- Frontend: Vite HMR updates in-browser without a full reload
- Both processes managed by the Supervisor

Before this fix, development required manual restarts after code changes, which significantly slowed iteration.
