## Overview

Two-factor authentication (TOTP) was added as an optional second layer on top of the portal password. A `--hosted` flag was introduced for running Fluxy as a managed hosted service.

---

## New Features

### Two-Factor Authentication (TOTP)

The onboarding wizard now offers TOTP-based 2FA. Once enabled:
- A QR code is shown during setup (compatible with Google Authenticator, Authy, etc.)
- On every login, users enter both their portal password and a time-based one-time code
- 2FA can be enabled/disabled through the onboarding wizard or settings

The implementation uses standard TOTP (RFC 6238) — any authenticator app works.

### Hosted Mode (`--hosted`)

A `--hosted` flag was added to the Fluxy CLI. When set, Fluxy behaves as a hosted/multi-tenant service rather than a single-user personal instance. The flag adjusts:
- Onboarding flow (no local path setup)
- Networking assumptions (expects to run behind a reverse proxy)
- Feature availability

### Handle Code Validation

Handle codes (the URL slug for each Fluxy instance, e.g. `fluxy.bot/yourname`) are now validated during onboarding to ensure uniqueness and format compliance before the wizard completes.
