## Overview

Fluxy now works as a proper background daemon on Linux (systemd) and macOS (launchd). Tailscale and named Cloudflare tunnels were added as networking options alongside the existing tunnel setup.

---

## New Features

### Linux Daemon (systemd)

Fluxy can now be installed and managed as a `systemd` service on Linux:

```bash
fluxy install    # installs as a systemd unit
systemctl start fluxy
systemctl enable fluxy  # auto-start on boot
```

The service file is generated by the CLI and placed in the appropriate systemd directory. Logs flow to `journald`.

### macOS Daemon (launchd)

On macOS, Fluxy installs as a `launchd` agent:

```bash
fluxy install    # installs as a LaunchAgent plist
launchctl load ~/Library/LaunchAgents/fluxy.plist
```

Auto-starts on user login. Logs go to `~/Library/Logs/fluxy.log`.

### Cross-Platform: Linux + Mac Verified

Both daemon modes were fully tested end-to-end on Ubuntu (Linux) and macOS. This was the first release where a fresh install on either platform resulted in a fully running, auto-starting Fluxy instance.

### Tailscale Integration

Tailscale was added as a networking option. When Tailscale is installed, Fluxy can expose itself over the Tailscale mesh network, giving you:
- Access from any device on your Tailnet
- No open ports on the public internet
- Works alongside other tunnel options

### Named Cloudflare Tunnels

Support for **named** Cloudflare tunnels was added (in addition to the temporary quick-tunnel mode). Named tunnels:
- Persist across restarts (same URL every time)
- Can be configured with a custom domain
- Are managed via `cloudflared tunnel` credentials
