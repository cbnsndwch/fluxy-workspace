# Workspace Improvements

A lightweight issue tracker for capturing things that need fixing, improving, or building in the workspace itself.

## Issue fields

- **Title** — short description of the issue
- **Description** — full details, steps to reproduce, proposed solution
- **Status** — `open`, `in-progress`, `done`, `wont-fix`
- **Priority** — `low`, `medium`, `high`, `critical`
- **Category** — `bug`, `feature`, `improvement`, `docs`, `other`

## Workflow

1. Spot something broken or improvable → create an issue
2. Assign a priority and category so it's findable
3. When working on it → set to `in-progress`
4. When resolved → `done`
5. If it's not worth fixing → `wont-fix`
