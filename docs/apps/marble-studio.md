# Marble Studio

Marble Studio is a text-to-3D world generator powered by the [World Labs](https://worldlabs.ai) Marble API. Describe a scene in words (or upload an image), and Marble Studio generates a fully navigable 3D Gaussian Splat world you can explore right in the browser.

---

## Getting Started

Before generating worlds, you need a World Labs API key.

1. Go to **Marble Studio → Settings** (gear icon in the sidebar)
2. Paste your World Labs API key
3. Hit **Save Key** — the key is encrypted with AES-256-GCM before being stored in the database

Once a key is saved, the Settings panel shows a masked hint (e.g. `sk-wl-••••••••abcd`) and a **Remove Key** option.

## Generating a World

Click **New World** in the gallery to open the generation form.

### Fields

| Field | Description |
|---|---|
| **World Name** | A label for your own reference — shown on the gallery card |
| **Model** | Which Marble model to use (see below) |
| **Prompt Mode** | How you'll describe the scene (see below) |
| **Prompt / Input** | The actual scene description or image URLs |

### Models

| Model | Notes |
|---|---|
| **Marble 1.1** | Current standard model — recommended for most generations |
| **Marble 1.1 Plus** | Larger, more detailed worlds — slower |
| **Marble 0.1 Mini** | Fast, legacy model |
| **Marble 0.1 Plus** | Legacy, higher quality |

### Prompt Modes

**Text** — describe the scene in natural language.
> *"A crystal cavern with glowing blue stalactites and an underground lake reflecting violet light"*

**Image** — provide a single image URL. Toggle **Panoramic** if the image is a 360° equirectangular panorama.

**Multi-Image** — up to 8 images with azimuth angles (0°–360°). Use this to reconstruct a scene from photos taken at different orientations.

**Video** — provide a URL to an MP4, MOV, or MKV file. Marble infers the 3D scene from the video.

**Presets** — choose from 16 curated prompts (Crystal Cavern, Volcanic Shore, Dragon Hoard, Sakura Garden, etc.) to get started immediately.

## Generation Flow

1. Submit the form → a **Generating** card appears in the gallery
2. The backend submits the request to World Labs and records an `operation_id`
3. The frontend polls `/api/marble-studio/worlds/:id/poll` every **5 seconds**
4. World Labs processes the generation (typically 1–5 minutes)
5. When done, the world's assets are saved: thumbnail, caption, and SPZ URLs at three resolutions
6. The gallery card flips to **Ready** with a thumbnail

If generation fails, the card shows an **Error** badge with the error message from World Labs.

## The Gallery

All your worlds live in a grid. Each card shows:
- Thumbnail (auto-generated by World Labs)
- World name
- Status badge: **Ready** / **Generating** / **Error**
- **Delete** button (trash icon, top-right of card)
- **Open in World Labs** button (external link icon) on ready worlds

Click a card to open the 3D viewer.

### Sync from World Labs

The **Sync** button (top of gallery) fetches your worlds directly from World Labs and backfills any that are missing locally. It also unsticks worlds that got stuck in "generating" status. Useful if you created worlds directly on worldlabs.ai.

## 3D Viewer

Opening a world loads a WebGL viewer in an iframe, powered by **Three.js** + **SparkJS** (Gaussian Splat renderer).

### Controls

| Action | Input |
|---|---|
| Rotate | Click + drag |
| Zoom | Scroll wheel / pinch |

A **hand drag hint** animates on first open — it disappears after 7 seconds or on your first interaction.

### Resolution Levels

Worlds are stored at three resolutions. The viewer automatically loads the best available quality:

| Level | Points | Notes |
|---|---|---|
| **100k** | ~100,000 splats | Loads first — fast |
| **500k** | ~500,000 splats | Auto-loaded silently after 300ms |
| **Full** | Full resolution | Manual — click the **Full** button |

Switching to a higher resolution triggers a **radial reveal transition**: the current frame freezes as an overlay, the new world loads underneath, and a circular hole expands from the center outward over ~3 seconds — mimicking the depth-sorted loading where near geometry resolves first.

The resolution buttons (100k / 500k / Full) in the top-right corner let you manually switch at any time.

### Download

The **Download SPZ** button (bottom of the viewer) downloads the current resolution's `.spz` file — the raw Gaussian Splat data. SPZ files can be opened in tools like [SuperSplat](https://superspl.at/).

## Data Model

```
marble_worlds
├── id            — local primary key
├── name          — display label
├── prompt        — the text/image/video input
├── prompt_type   — 'text' | 'image' | 'multi-image' | 'video' | 'presets'
├── model         — which Marble model was used
├── world_id      — World Labs world ID (set when generation completes)
├── operation_id  — World Labs operation ID (used for polling)
├── status        — 'pending' | 'generating' | 'done' | 'error'
├── error_msg     — error from World Labs (if status = error)
├── assets_json   — JSON blob with SPZ URLs at each resolution + thumbnail
├── thumbnail_url — direct thumbnail URL
├── caption       — auto-generated caption from World Labs
└── created_at    — creation timestamp

marble_studio_settings
├── key           — setting name ('api_key', '__encrypt_key__')
└── value         — encrypted value (AES-256-GCM)
```

---

## Security

Your World Labs API key is **never stored in plaintext**. On first save:
1. A random 256-bit encryption key is generated and stored as `__encrypt_key__` in `marble_studio_settings`
2. Your API key is encrypted with AES-256-GCM (random IV + authentication tag) and stored as `api_key`
3. The encrypted format is: `enc:<iv_base64>.<tag_base64>.<ciphertext_base64>`

The encryption key lives in the same database as the encrypted value — this protects against casual exposure but not against full database access. Keep your `app.db` private.

## API Reference

| Method | Path | Description |
|---|---|---|
| `GET` | `/api/marble-studio/settings` | Check if API key is set (returns hint, not the key) |
| `PUT` | `/api/marble-studio/settings` | Save a new API key |
| `DELETE` | `/api/marble-studio/settings` | Remove the stored API key |
| `GET` | `/api/marble-studio/worlds` | List all worlds |
| `POST` | `/api/marble-studio/worlds` | Create + submit a new world |
| `GET` | `/api/marble-studio/worlds/:id` | Get a single world |
| `GET` | `/api/marble-studio/worlds/:id/poll` | Poll generation status (called every 5s) |
| `DELETE` | `/api/marble-studio/worlds/:id` | Delete a world |
| `POST` | `/api/marble-studio/worlds/sync` | Sync worlds from World Labs |
| `GET` | `/api/marble-studio/viewer` | Returns the 3D viewer iframe HTML |
