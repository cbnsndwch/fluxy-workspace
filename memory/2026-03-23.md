## Workflow Engine — Major Feature Sprint

Diego asked to explore n8n feature parity and fix gaps. Implemented:

### Sandboxing (was already done, documented)
- Two-layer: Worker thread (separate V8 heap) + vm.runInNewContext inside
- Only pure JS built-ins exposed (Math, Date, JSON, Array, Object, Map, Set, etc.)
- No require/fs/process/network access
- 5s outer timeout (worker.terminate) + 4s inner timeout (vm sync loops)
- Data crosses boundary via JSON round-trip (structured clone)

### Variable interpolation (NEW)
- Added `interpolate(template, ctx)` helper supporting `{{key}}` and `{{key.nested.path}}`
- Applied to: http_request URL, headers, body — ALL support context interpolation now
- Log node already had it, now consistent everywhere

### New node types
1. **cron_trigger** — Rose/pink. Has schedule (cron expression) + initial_data. Scheduler polls every 30s, deduplicates by minute key.
2. **transform** — Orange. Single JS expression (no `return` needed). `input.price * 1.21`, `Object.keys(input)`, etc. Runs in sandbox.
3. **db_query** — Teal. SQL query against workspace SQLite. SELECT returns row array, INSERT/UPDATE/DELETE returns {changes, lastInsertRowid}. Supports `?` placeholders via params array. Both query and params support {{interpolation}}.

### Cron scheduler
- Starts automatically when backend starts (inside createRouter)
- Uses CronExpressionParser.parse() from cron-parser v5 (note: no parseExpression named export in v5, use CronExpressionParser.parse)
- Checks prev() occurrence against current time with 60s window

### DB connection
- Already global/shared singleton — no connection overhead per node execution
- db_query node passes the existing db instance through executeNode signature
