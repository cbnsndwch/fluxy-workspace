## Marketplace App — Full implementation (2026-04-07)

Built the Marketplace app end-to-end:

### Backend (`backend/apps/marketplace/index.ts`)
- Tables: marketplace_tokens, marketplace_orders, marketplace_settings, marketplace_error_reports, marketplace_telemetry
- Routes:
  - GET /api/marketplace/apps — catalog
  - POST /api/marketplace/checkout — creates order + short-lived UUID token
  - GET/DELETE /api/marketplace/tokens — list + revoke
  - GET /api/marketplace/orders — order history
  - POST /api/marketplace/redeem — validate/redeem token (buyer)
  - GET /api/marketplace/bundle/:appId — bundle download stub (requires valid token)
  - GET/PUT /api/marketplace/settings — opt-in feature settings
  - POST /api/marketplace/report-error — receive error reports from buyer apps
  - POST /api/marketplace/telemetry — receive telemetry events from buyer apps
  - GET /api/marketplace/error-reports — view incoming errors
  - GET /api/marketplace/telemetry-events — view telemetry
  - GET /api/marketplace/stats — dashboard stats

### Frontend (`client/src/components/Marketplace/MarketplacePage.tsx`)
Full AppLayout + 4-tab rewrite:
- **Store** — Tier picker (Solo/Starter/Pro/Everything) + 12-app grid with slot-limited selection + checkout bar
  - CheckoutDialog: buyer email, notes, expiry selector (24h/48h/7d) → generates token
  - Token display: copyable UUID + install command + expiry countdown + app pills
- **Tokens** — Table of all generated tokens with status (active/expired/redeemed/revoked), copy install command, revoke button
- **Reports** — Error reports viewer + telemetry events viewer (gated by settings)
- **Settings** — Error tracking toggle, telemetry toggle, external API forwarding toggle + URL input, SDK usage snippet, workspace ID display

### Telemetry SDK (`client/src/lib/appTelemetry.ts`)
- `reportError(appId, error, context?)` — sends to /api/marketplace/report-error
- `trackEvent(appId, eventType, payload?)` — sends to /api/marketplace/telemetry
- Caches settings for session, invalidatable
- All functions are silent no-ops when not opted in — never throw, never affect app behavior
- `invalidateTelemetryCache()` exported for settings change scenarios

### Token flow
1. Diego selects tier + apps → clicks "Generate token"
2. Optional: buyer email, notes, expiry (24h/48h/7 days)
3. Token generated (UUID) → dialog shows token + install command
4. Buyer tells their Fluxy: "install bundle [TOKEN]"
5. Fluxy calls POST /api/marketplace/redeem → validates, marks redeemed, returns app list
6. Each app can be downloaded via GET /api/marketplace/bundle/:appId?token=TOKEN
