## ~09:00 — Dedup Workflow Enhancements
- **Execute & Continue**: Summary screen now has two buttons — "Execute & Close" (original) and "Execute & Continue" (green). Continue mode rescans at current threshold after merging; if empty, auto-steps down to next threshold with results.
- **Available thresholds distribution**: Backend now computes which 1% thresholds have clusters in a single pass (all pairwise similarities above 50% computed once, then clustered at each step). Returned as `available_thresholds[]` alongside clusters.
- **"Find Next ↓" button**: Appears next to Rescan when there's a lower threshold with results. Shows the exact percentage. Also appears on empty state when no clusters found at current threshold.
- **Next threshold hint in summary**: Before execute buttons, shows "Next threshold with results: X%" so Diego knows what's coming.
- Diego's workflow: start high (98%), review most-certain merges, execute & continue, tool auto-drops to next productive threshold. Rinse until done.

### Dedup Dismissal Persistence
- **Problem**: "Not Duplicates" decisions weren't saved — every rescan resurfaced the same pairs Diego already rejected. Massive friction when ratcheting down similarity thresholds.
- **Solution**: New `onto_dedup_dismissals` table stores node ID pairs (normalized as min/max). All pairwise combos from a "Not Duplicates" group are saved.
- **Backend changes**: `findDuplicateClusters` loads dismissals into a Set and skips those pairs during similarity computation. Merge endpoint now accepts `dismissals[]` alongside `merges[]`. New `DELETE /api/ontologica/projects/:id/dismissals` to reset if needed.
- **Frontend changes**: "Not Duplicates" decision now sends `dismissals` array in the execute payload. Toast confirms how many groups were dismissed. "Reset dismissals" link in dialog header (with confirmation) clears all dismissals and rescans.
- Dismissals survive across sessions — once you say "not duplicates", those pairs are gone until you explicitly reset.

### Split Mode UX Overhaul
- **Drag & drop**: Items in split mode are now draggable between Group A and Group B using HTML5 drag API. Drop zones highlight on hover (violet for A, blue for B). `cursor-grab` / `cursor-grabbing` states.
- **Click to pick keeper**: Each group now has independent canonical selection. Click a node card → green checkmark + "keep" label. Group A uses `canonicalId`, Group B uses `splitBCanonical`. Previously there was no way to pick which item to keep per group.
- **Smart canonical management**: When dragging the canonical node out of a group, auto-selects the next most-connected node. First node dropped into Group B auto-becomes its canonical.
- Separation of concerns: drag = move between groups, click = select keeper within a group.

- Created `expandSchemaOrg()` function since `seedBaseLayers()` is guarded by existing count check — expansion runs on every backend boot, INSERT OR IGNORE makes it idempotent

### Fix 2: Hybrid Matching (Embeddings + LLM)
- Old: pure embedding similarity (all-MiniLM-L6-v2) — can't reason about IS-A relationships
- New: two-phase approach:
  1. Embeddings with lower threshold (35%) to cast a wide net → top-5 candidates per node
  2. GPT-4o-mini evaluates all candidates semantically → returns match_type (same/is_a/related/no_match) + confidence
- Fixed embedding text: strip namespace prefixes (schema:AccountingService → AccountingService), split camelCase into words (AccountingService → Accounting Service)
- LLM batches pairs in groups of 40 per API call
- Falls back to embedding-only if no OPENAI_API_KEY or LLM fails

### Frontend Updates
- Match type badges: "Same as" (emerald), "Is a type of" (amber), "Related to" (blue)
- Recommended action highlighted: "Link to base" solid green for same, "Type of X" solid amber for is_a
