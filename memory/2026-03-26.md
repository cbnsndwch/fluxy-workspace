## 16:40 — Flow Capture: persistent chunks + sessions system

Diego wanted the speech-to-diagram app upgraded: each speech fragment should be an independent persistent item so the diagram can be reconstructed and remixed later.

**Built:**
- Backend: `flow_sessions`, `flow_chunks`, `flow_diagrams` SQLite tables
- REST API: CRUD for sessions, chunk save/delete, diagram generation via OpenAI gpt-4o-mini
- `Remix` mode: regenerates diagram from all chunks without using current diagram as base
- Frontend: session picker dropdown, chunks shown as numbered persistent cards, live interim text, delete individual chunks, error healing (keeps last valid SVG)
- Backend `/flow-capture/sessions/:id/diagram` accepts `remix: true` for fresh generation
- Backend updated to pass `db` to flow-capture router

Uses OpenAI (no Anthropic key in .env). Diagram heals by keeping previous SVG when mermaid parse fails.

## 21:26 — Issue #21: Flow Capture segment selection + diagram highlighting

Built two features:
1. **Freshly-added segments** get a violet ring/dot after each diagram generation. Tracks `pendingChunkIdsRef` → captured into `freshChunkIds` on generation start, set after success. Clears on next generation cycle or session load.
2. **Click a card → select it → highlight diagram nodes**: `selectedChunkSeq` state drives `highlightedSvg` useMemo that injects CSS `<style>` into the SVG targeting `g[id^="flowchart-{nodeId}-"]` elements with violet fill + glow.
3. Separated edit (pencil icon) from select (card click). Click card body → select/deselect. Click pencil → edit modal.
4. Backend: diagram generation now returns `chunkNodeMap` (JSON object, chunk sequence → node IDs). Stored in `flow_diagrams.chunk_node_map` column. Returned on session load.
5. Diagram prompt switched to JSON response format with `{ "diagram": "...", "chunkNodeMap": {...} }`.
