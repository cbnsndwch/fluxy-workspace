
## Issue #52 — Uploads app: useNavigate router error in toast [CRON]
Fixed by moving `useNavigate()` from `IssueCreatedToast` to `ReportIssueAction` and passing it as a prop. Root cause: Sonner's `toast.custom()` renders components outside the React Router tree, so any hook depending on Router context (like `useNavigate`) must be called in the parent that *is* inside the Router. Branch: `issue/52`.

## Issue #51 — Uploads lightbox: incorrect image href [CRON]
The listing endpoint was returning URLs of the form `/app/api/uploads/image/:filename`, but the correct authoritative path is `/api/files/images/:filename` (served by the worker's express.static middleware). Changed line 60 in `backend/apps/image-gen/index.ts` to use `/api/files/images/${filename}`. Fix already committed on branch `issue/51` by a previous run — detected, verified, and reported done.
